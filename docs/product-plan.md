# 产品规划

## 一句话定义

一个用于铝型材定制结构的微信小程序：输入尺寸，生成 3D 结构，输出材料和配件清单。

## 版本节奏

## V0.1：需求和技术骨架

目标：把项目从旧项目切换成铝型材结构设计器。

交付：

- README
- 功能需求清单
- 数据模型
- BOM 计算规则
- 小程序目录骨架

## V0.2：模板生成 MVP

目标：用户可以选择模板并生成基础结构。

功能：

- 首页项目列表
- 模板选择
- 输入宽 / 高 / 深
- 生成 nodes / beams / panels
- 本地保存项目

模板优先级：

1. 书桌
2. 衣柜
3. 置物架
4. 电视支架

## V0.3：3D 预览

目标：用户能看到结构比例，而不是只看清单。

功能：

- Canvas 3D 简化渲染
- 拖动旋转
- 双指缩放
- 正视 / 侧视 / 俯视
- 杆件高亮

## V0.4：BOM 清单

目标：生成可用于采购和加工沟通的清单。

功能：

- 型材切割清单
- 连接件清单
- 板材清单
- 损耗率设置
- 复制 BOM 文本

## V0.5：编辑器增强

目标：从模板工具升级为轻量结构编辑器。

功能：

- 添加横梁
- 添加竖梁
- 删除杆件
- 调整层数 / 分区
- 节点吸附
- 单根杆件参数编辑

## V0.6：报价与采购

目标：让设计清单变成采购决策。

功能：

- 型材单价维护
- 配件单价维护
- 板材单价维护
- 加工费配置
- 总价估算

## MVP 边界

第一版不做：

- 用户登录
- 云端同步
- 多人协作
- 复杂 CAD 自由曲线
- 真实受力仿真
- 供应商自动下单

第一版只打穿一条主链路：

> 模板 → 尺寸 → 结构 → 3D → BOM

## 产品判断

这个项目真正有价值的地方，不是把小程序做成 CAD，而是把装修定制里最容易扯皮的部分前置固化：

- 尺寸是否一致
- 型材规格是否一致
- 连接件是否包含
- 切割长度是否清楚
- 板材是否另算
- 损耗和加工是否另算

它应该成为一张会发光的报价单，也是一套能落地的结构语言。
